# GROUPBEE REACT

## ☘️ 1. 로그인
<img src="https://velog.velcdn.com/images/woong2/post/dac868da-f556-4d1a-85fe-fba839ebf716/image.png" style="width: 500px; height: auto;" alt="로그인 이미지">

[Zustand](https://zustand-demo.pmnd.rs/) 상태 관리 라이브러리를 사용하여 서버로 아이디와 패스워드를 전송하고, 입력한 정보가 올바를 경우 **세션을 발급받아** 로그인 상태를 유지하는 방식을 사용했습니다.

### 사용 기술
- **Zustand**: 상태관리 라이브러리. 


### 참고 자료
- [Zustand 공식 문서](https://zustand-demo.pmnd.rs/)

### 주요 기능


#### 1-1. 상태코드
아이디와 패스워드를 서버로 전송한 후, 입력한 값의 유효성에 따라 상태 코드를 반환하여 오류 메시지를 구분합니다.

+ 상태코드 200 : 로그인 성공
+ 상태코드 401 : 존재하지 않는 ID입니다.
+ 상태코드 402 : 비밀번호가 틀렸습니다.


#### 1-2. 새로고침 시 상태관리 유지
로그인 상태를 Zustand 저장소에 저장 했을 경우 **새로고침 시 저장된 상태가 초기화되는 문제가 발생**하기 때문에, 새로고침 후에도 로그인 상태를 유지하기 위해 **발급된 세션을 다시 서버로 전송하여 유효한 세션이면 상태 코드 200을 반환**받고, 이를 통해 로그인 상태를 유지하는 방식을 사용하고 있습니다.

## ☘️ 2. 메인페이지
<img src="https://velog.velcdn.com/images/woong2/post/542c8d1a-2211-4047-bac0-e486b10d9420/image.jpeg" style="width: 500px; height: auto;">
 
 React 기반으로 설계된 웹 애플리케이션으로, 회사의 주요 정보를 한눈에 확인할 수 있습니다. 사용자에게 공지사항, 결재 현황, 예약 상태, 날씨 정보, 캘린더 등의 다양한 정보를 직관적으로 제공합니다.

### 사용 기술 
- **React (hooks 사용)** 
- **Material-UI** 
- **Axios** 
- **FullCalendar** 
- **SweetAlert2**
- **OpenWeather**

### 주요 기능 
#### 2-1. 공지사항 
- 최신 공지사항을 표시하며, 중요 공지는 강조 표시됩니다.  
- 각 공지는 제목, 작성자, 작성일을 포함합니다. 

#### 2-2. 출퇴근 기능 
- 로컬 시간대를 기준으로 현재 시간을 출력합니다.
- 출근 및 퇴근 버튼을 통해 근태를 체크할 수 있습니다. 
- 당일의 출근 및 퇴근 시간이 표시됩니다. 

#### 2-3. 날씨 정보 
- OpenWeather API를 사용해 현재 장소에 대한 날씨 정보를 제공합니다.
- 온도, 날씨 상태, 습도 등의 정보를 제공합니다. 

#### 2-4. 결재 현황 
- 사용자의 결재 문서 상태를 확인할 수 있습니다. 
- 상태별(전체, 반려, 결재대기, 결재중, 결재완료) 필터링이 가능합니다. 
- 각 문서의 종류, 제목, 작성자, 상태 정보를 표시합니다. 

#### 2-5. 예약 현황 
- 회의실이나 차량 등의 예약 상태를 보여줍니다. 
- `DashboardBook` 컴포넌트를 통해 구현되어 있습니다. 

#### 2-6. 캘린더 
- 월간 일정을 확인할 수 있는 캘린더 기능을 제공합니다. 
- `FullCalendar` 라이브러리를 사용하여 구현되었습니다. 
- 일정 클릭 시 상세 정보를 확인할 수 있습니다.
## ☘️ 3. 조직도
<img src="https://velog.velcdn.com/images/woong2/post/19039273-f797-42fe-9278-7f5988f53467/image.png" style="width: 500px; height: auto;">
이 컴포넌트는 부서 구조와 직원 정보를 표시하고 선택할 수 있는 인터페이스를 제공합니다. 조직도는 상위, 중간, 하위 부서로 구분되어 있으며, 사용자는 각 부서를 클릭해 해당 부서와 그 하위 부서에 속한 직원 목록을 확인할 수 있습니다. 이 기능은 백엔드 없이 프론트엔드에서만 구현되었습니다.

### 사용 기술
- **React**: 프론트엔드 프레임워크.
- **Axios**: API 요청을 위한 라이브러리.
- **CSS**: 스타일링을 위한 기본 CSS.

### 주요 기능 
#### 3-1. 부서 목록 표시 
- 부서는 상위, 중간, 하위 부서로 나뉘며, 트리 구조로 화면에 표시됩니다. 
- 각 부서를 클릭하면 해당 부서와 관련된 직원 목록이 오른쪽에 표시됩니다.

#### 3-2. 직원 목록 필터링 
- 부서를 클릭하면 해당 부서에 속한 직원들이 필터링되어 화면에 표시됩니다. 
- "하위 부서 포함" 체크박스를 선택하면 선택된 부서의 하위 부서에 속한 직원들도 함께 표시됩니다. 

#### 3-3. 직원 선택 
- 각 직원은 체크박스를 통해 선택할 수 있으며, 선택한 직원들은 `selectedEmployees` 상태에 저장됩니다. 
- "해당 부서 전체 선택" 체크박스를 통해 한 번에 해당 부서의 모든 직원을 선택하거나 해제할 수 있습니다. 

#### 3-4. 조직도 확장/축소 
- 상위 부서, 중간 부서를 클릭하여 해당 부서의 하위 부서를 확장하거나 축소할 수 있습니다. 클릭된 부서는 열리고, 다른 부서는 자동으로 닫힙니다. 

### 상태 관리 
- **부서 리스트**: `departmentList` 상태에 API로부터 불러온 부서 목록이 저장됩니다. 
- **직원 리스트**: `employeeList` 상태에 모든 직원 목록이 저장됩니다. 
- **선택된 부서**: `selectedDepartment`와 `selectedDepartmentNames` 상태에 선택된 부서와 그 부서의 이름 목록이 저장됩니다. 
- **필터링된 직원**: `filteredEmployees` 상태에 선택된 부서(및 하위 부서)에 속한 직원들이 저장됩니다. 
- **선택된 직원**: `selectedEmployees` 상태에 현재 선택된 직원들이 저장됩니다. 
- **하위 부서 포함 여부**: `includeSubDepartments` 상태에 하위 부서를 포함할지 여부가 저장됩니다.

## ☘️ 4. 전자결재
<img src="https://velog.velcdn.com/images/woong2/post/47de0bd5-97ba-45a9-b9c9-cd535f60af18/image.png" style="width: 500px; height: auto;">

전자결재 시스템은 문서 작성, 결재 요청 및 처리 과정을 한층 간편하고 효율적으로 관리할 수 있는 스마트 웹 애플리케이션입니다. 디지털화를 통해 결재 프로세스를 혁신하여 사용자 편의성을 극대화했습니다. 복잡한 결재 절차는 이제 그저 클릭 몇 번으로 해결하세요!

### 사용된 기술
- **React**
- **Material-UI**

### 주요 기능
- **결재 요청 생성**: 간단한 양식으로 신속하게 결재 요청을 생성할 수 있습니다.
- **기본 정보 호출**: OpenFeign을 활용하여 사용자의 이름, 부서, 직급 등의 정보를 자동으로 입력해줍니다.
- **찾기 버튼**: 조직도와 연동되어 직원 선택이 간편해졌습니다.
- **결재자 관리**: 선택할 수 있는 결재자 모달이 있으며, 중복 승인자나 본인 선택 방지 로직이 포함되어 있어 효율적입니다.
- **결재 처리**: 결재자는 요청을 실시간으로 승인하거나 반려하며, 상태가 즉시 반영됩니다.
- **폼 검증**: 필수 입력 항목을 체크하고 오류 메시지를 통해 사용자의 실수를 방지합니다.
- **임시 저장**: 작성 중인 서류를 임시로 저장할 수 있어 언제든지 다시 시작할 수 있습니다.
- **파일 업로드**: 드래그 앤 드롭 또는 파일 선택 버튼을 통해 파일을 손쉽게 업로드할 수 있습니다.

### 사용 방법
- **결재 요청 생성**: 결재 요청을 생성하고 필요한 정보를 입력하세요.
- **결재자 선택**: 조직도에서 결재자를 선택하고 요청을 제출합니다.
- **결재 확인 및 처리**: 결재자는 대시보드에서 요청을 확인하고 결재를 승인하거나 반려할 수 있습니다.

### 추가 기능
- **상단 필드 고정**: 결재 요청 목록에서 상단 필드를 고정하여 더 효율적으로 결재 요청을 관리할 수 있습니다.
- **하단 필드 수정**: 하단 필드는 쉽게 수정 가능하며, 다양한 문서 형식을 추가하는 것이 간편합니다.

### 구현 세부 사항
- **결재 요청 목록 페이지**
    - 결재 상태에 따른 필터링 기능
    - 페이지네이션 구현으로 효율적인 데이터 로딩
    - 결재 상태에 따른 색상 구분으로 직관적인 UI 제공
- **결재 상세 페이지**
    - 문서 유형(품의서, 휴가신청서, 지출보고서)에 따른 동적 렌더링
    - 결재 단계별 승인 상태 시각화
    - 반려 사유 모달 구현으로 상세 정보 제공
- **발신 목록 페이지**
    - 사용자가 작성한 문서의 상태 관리
    - 임시저장 문서 접근 및 편집 기능
- **공통 기능**
    - 날짜 포맷팅을 통한 일관된 날짜 표시
    - 첨부 파일 다운로드 기능 구현
    - 결재 승인 및 반려 효율적인 관리

## ☘️ 5. 캘린더
<img src="https://velog.velcdn.com/images/woong2/post/ed24fabc-8d44-4972-9e37-9ed0414e85f3/image.png" style="width: 500px; height: auto;">
캘린더는 대한민국 공휴일, 사용자가 추가한 일정, 그리고 회사 내 자산 예약 정보를 포함한 총 3가지 주요 기능을 제공합니다. 또한, 오늘 날짜의 일정들을 관리할 수 있는 To-do 리스트 기능도 함께 제공합니다.

### 사용 기술
- **Redis**
- **FullCalendar**

### 참고 자료 
- [Redis 공식 문서](https://redis.io/enterprise/?gad_source=1&gclid=Cj0KCQjwo8S3BhDeARIsAFRmkOM6o1GdtOPsTQIfbXek25XaGcOm2FR_qOf5LKk6u4_U8gv_xLPXpDkaAkoREALw_wcB&utm_campaign=gg_s_brand_bam_acq_apac-en_21161918358&utm_source=google&utm_medium=cpc&utm_content=redis_exact&utm_term=)
- [FullCalendar 공식 문서](https://fullcalendar.io/)

### 주요 기능

#### 5-1. 트랜잭션 처리
트랜잭션 처리를 통해 데이터 무결성을 확보하여 예약 충돌이나 데이터 불일치 문제를 방지할 수 있습니다. 이를 통해 여러 사용자가 동시에 예약할 때도 일관된 데이터 상태를 유지할 수 있습니다.

#### 5-2. 한국 공휴일 출력
기존의 풀캘린더(FullCalendar)에 구글 캘린더 API를 연동하여 대한민국 공휴일을 자동으로 출력하도록 구현하였습니다. 이를 통해 사용자는 공휴일 정보를 실시간으로 확인할 수 있습니다.

#### 5-3. To-do 리스트 기능
오늘의 일정 섹션을 통해 사용자는 오늘 날짜에 해당하는 일정 목록을 한눈에 확인할 수 있습니다. 이를 통해 중요한 일정이나 작업을 놓치지 않고, 효율적으로 하루를 관리할 수 있습니다.




## ☘️ 6. 예약
<img src="https://velog.velcdn.com/images/woong2/post/8e7c5a50-967c-4308-837a-466b6c261bdd/image.jpeg" style="width: 500px; height: auto;">
예약은 기본 CRUD를 바탕으로 차량/회의실의 내역을 확인하고 실시간 예약을 할 수 있습니다.
MUI나 페이지네이션 등 다양한 react 기반의 UI 컴포넌트를 사용하여 구성했습니다.

### 사용 기술
- **Material-UI**
- **Bootstrap**

### 참고 자료
- [Material-UI 공식 문서](https://mui.com/)
- [Bootstrap 공식 문서](https://getbootstrap.kr/)

### 주요 기능

#### 6-1. 프로그래스바
1시간 단위로 24개로 나눈 프로그래스바를 통해 예약 현황을 실시간으로 확인할 수 있으며, 현재 상태를 한눈에 파악할 수 있습니다. 이 프로그래스바는 예약이 가능한 상태에서는 노란색으로 표시되고, 예약이 마감된 상태는 회색으로 나타납니다.

시간 선택 시 이미 예약된 시간대는 실시간으로 처리하여 프론트엔드에서 선택할 수 없게 구성하였습니다. 예약하려는 시간대 사이에 이미 예약자가 있을 경우, 시간 선택 후 예약이 불가능하도록 처리하고 경고 메시지를 출력하도록 하였습니다.



## ☘️ 7. 게시판

## ☘️ 8. 이메일
<img src="https://velog.velcdn.com/images/woong2/post/18222e8f-66ce-4402-b5e3-572ebab4c122/image.png" style="width: 500px; height: auto;">

이메일 시스템은 기업 내 커뮤니케이션을 원활하게 하는 핵심 툴입니다. 간단한 메일 작성부터 받은 메일 관리까지, 이메일 관련 모든 작업을 한 번에 처리할 수 있으며, 외부 메일과도 통신이 가능합니다.

### 사용된 기술
- **React**
- **Material-UI**
- **axios**

### 주요 기능
- **메일 작성**: 직관적인 인터페이스로 쉽고 빠르게 메일을 작성할 수 있습니다.
- **조직도 연동**: 조직도와 연동되어 직원 선택이 간편해졌습니다.

## ☘️ 10. 채팅

실시간 채팅 시스템은 기업 내 즉각적인 커뮤니케이션을 가능하게 하는 핵심 도구입니다. 사용자 친화적인 인터페이스와 실시간 메시징 기능을 결합하여 팀원 간의 효율적인 소통을 지원합니다. 1:1 채팅부터 그룹 채팅까지, 다양한 형태의 대화를 원활하게 진행할 수 있습니다.

### 사용된 기술
- **React**
- **WebSocket**
- **Axios**
- **Material-UI**

### 주요 기능
- **실시간 메시징**: WebSocket을 이용한 즉각적인 메시지 전송 및 수신
- **1:1 및 그룹 채팅**: 개인 간 대화 및 다수의 참여자와의 그룹 채팅 지원
- **채팅방 생성 및 관리**: 새로운 채팅방 생성 및 기존 채팅방 관리 기능
- **참여자 관리**: 채팅방 참여자 목록 확인
- **검색 기능**: 채팅방 검색

### 사용 방법
- **채팅방 생성**: '+' 버튼을 클릭하여 새로운 채팅방 생성
- **메시지 전송**: 채팅창 하단의 입력란에 메시지 입력 후 전송 버튼 클릭 또는 Enter 키 입력
- **채팅방 관리**: 각 채팅방의 '⋯' 버튼을 클릭하여 채팅방 나가기 또는 이름 변경 등의 관리 기능 사용
- **채팅방 검색**: 상단 검색 창을 통해 채팅방 이름으로 검색

### 구현 세부 사항
#### 채팅 목록 (Sidebar)
- 채팅방 목록 실시간 업데이트
- 최근 메시지 및 시간 표시
- 채팅방 검색 및 필터링 기능
#### 채팅방 (ChatRoomContainer)
- WebSocket을 통한 실시간 메시지 송수신
- 메시지 발신자에 따른 UI 구분 (왼쪽/오른쪽 정렬)
- 날짜 변경 시 날짜 구분선 표시
#### 채팅 관리 기능
- 채팅방 나가기 및 전체 나가기 기능
- 채팅방 이름 변경 기능
#### 사용자 경험 개선
- 프로필 사진 및 이름 표시로 발신자 구분 용이
- 메시지 전송 시간 표시

## ☘️ 11. OpenVidu 화상회의

이 화상 회의는 [OpenVidu](https://openvidu.io/)를 사용해 커스텀 컨트롤바 및 채팅 기능을 제공하며 프로젝트에 맞춘 인터페이스를 구성했습니다. 컨트롤바와 채팅 UI를 커스터마이징했습니다.

### 사용 기술
 - **OpenVidu**: 실시간 비디오 스트리밍 솔로션. 


### 참고 자료 
- [OpenVidu 공식 문서](https://docs.openvidu.io/en/stable/) 


### 주요 기능
 - **커스텀 컨트롤바**: 프로젝트에 맞춘 비디오 컨트롤바.
 - **커스텀 채팅**: 그룹메시지를 지원하는 맞춤형 채팅 기능. 
- **반응형 영상**: 채팅창을 열고 닫을 때마다 화면 크기가 자동으로 조정되는 반응형 비디오.
- **비디오/오디오제어**: 사용자가 비디오와 오디오를 손쉽게 켜고 끌 수 있는 기능.
- **실시간 스트리밍**: OpenVidu를 통한 고품질 실시간 화상 회의.

### 사용 방법 
- **룸 참가**: 회의에 참가하려면 룸 이름을 입력하고 시작할 수 있습니다. 초대 코드를 다른 사용자와 공유해 손쉽게 초대할 수 있습니다.
- **컨트롤바**: 컨트롤바에는 다음과 같은 기능이 포함됩니다: 
- 마이크 음소거/음소거 해제 
- 카메라 켜기/끄기 
- 회의 종료
 - **채팅**: 채팅 아이콘을 클릭하여 채팅 창을 열거나 닫을 수 있습니다. 그룹 메시지 기능을 통해 참여자들과 대화를 나눌 수 있습니다.

### 커스터마이징

-**컨트롤바**: OpenVidu의 기본 컨트롤바는 회의 종료 후에도 화면이 그대로 남아 있어 사용자 경험이 불편할 수 있기 때문에, 처음부터 직접 구현하였습니다.
- 비디오/오디오 : OpenVidu에서 제공하는 기본 컴포넌트를 재사용하여 커스터마이징했습니다.
- 나가기버튼 : 새로 구현한 나가기 버튼은 룸 이름과 토큰 값을 null로 설정해 회의를 완전히 종료하도록 설계했습니다.
- **채팅**: 기본 채팅 기능은 다수의 참여자 중 누가 메시지를 보냈는지 확인이 어려웠기 때문에, 사용자 이름을 불러와 메시지 상단에 표시하도록 변경했습니다. 메시지 시간은 마지막 메시지나 1분이 지난 후, 또는 다른 참여자가 새로운 메시지를 보낼 때 업데이트되도록 구현했습니다. 또한, 내가 보낸 메시지는 오른쪽에, 상대방의 메시지는 왼쪽에 표시되도록 커스터마이징했습니다.
